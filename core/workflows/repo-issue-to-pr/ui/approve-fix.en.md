short_description: Waits for you to approve the implementation.
long_description: Pauses the workflow and asks you to review the implementation for the issue. Approve to continue to the pull request, or reject with notes to have it revised.
---
Pauses here so a person can look over the implementation before a pull request is opened. You can approve it, or reject with a short note on what should change — the workflow will revise the implementation and try again, up to a couple of tries.
