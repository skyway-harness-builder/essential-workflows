long_description: "Start zodra een issue het label \"sky-spec\" krijgt: schrijft een plan, wacht op jouw goedkeuring, implementeert het, wacht opnieuw op goedkeuring, en opent daarna een pull request."
---
Deze workflow neemt een groter stuk werk — een spec — en brengt het helemaal tot een pull request, met twee controlemomenten voor jouw inbreng onderweg. Eerst schrijft hij een plan dat beschrijft wat er verandert en waarom, en wacht op jouw goedkeuring. Zodra je het plan goedkeurt, implementeert hij het en draait de tests, en wacht daarna opnieuw op jouw goedkeuring voordat er een pull request wordt geopend. Wijs je een van beide keren af, dan wordt jouw feedback meegenomen en die stap opnieuw geprobeerd.
