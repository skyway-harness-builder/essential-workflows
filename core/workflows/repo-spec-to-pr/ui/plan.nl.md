long_description: Leest de spec en stelt een gestructureerd plan op met welke bestanden veranderen, de aanpak en risico's, en de teststrategie — nog zonder code te schrijven.
---
Leest de spec door en stelt een plan op voor de implementatie: welke bestanden moeten veranderen en waarom, wat er mis zou kunnen gaan, en hoe het getest wordt. In deze fase wordt er nog geen code geschreven — enkel een plan, zodat je de aanpak kunt beoordelen voordat het werk begint.
